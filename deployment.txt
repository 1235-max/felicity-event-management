# Felicity Event Management - Deployment Links

### Frontend (Vercel)
**Production URL**: https://felicity-frontend.vercel.app
- Platform: Vercel
- Framework: React + Vite
- Build Command: `npm run build`
- Output Directory: `dist`

### Backend (Render)  
**Base API URL**: https://felicity-backend.onrender.com
- Platform: Render
- Runtime: Node.js
- Start Command: `npm start`

### Database
**MongoDB Atlas**: Connected via MONGODB_URI environment variable

## 📦 Deployment Instructions

### Prerequisites
1. GitHub account with repository pushed
2. MongoDB Atlas account with database created
3. Vercel account (for frontend)
4. Render account (for backend)

---

### 🎨 Frontend Deployment (Vercel)

#### Step 1: Prepare Repository
```bash
cd frontend
git add .
git commit -m "Prepare for deployment"
git push origin main
```

#### Step 2: Deploy to Vercel
1. Go to https://vercel.com/new
2. Import your GitHub repository
3. Configure project:
   - **Framework Preset**: Vite
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Install Command**: `npm install`

#### Step 3: Environment Variables
Add the following in Vercel Dashboard → Settings → Environment Variables:
```
VITE_API_URL=https://your-backend-url.onrender.com/api
```

#### Step 4: Deploy
- Click "Deploy"
- Wait for build to complete
- Your frontend will be live at `https://your-project.vercel.app`

---

### ⚙️ Backend Deployment (Render)

#### Step 1: Prepare Repository  
Ensure `render.yaml` exists in backend folder (already configured)

#### Step 2: Create MongoDB Atlas Database
1. Go to https://cloud.mongodb.com
2. Create a new cluster (free tier available)
3. Create a database user with password
4. Whitelist IP: `0.0.0.0/0` (allow all IPs)
5. Get connection string: 
   ```
   mongodb+srv://username:password@cluster.mongodb.net/felicity?retryWrites=true&w=majority
   ```

#### Step 3: Deploy to Render
1. Go to https://render.com/create
2. Select "Web Service"
3. Connect your GitHub repository
4. Configure:
   - **Name**: felicity-backend
   - **Root Directory**: `backend`
   - **Environment**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Plan**: Free

#### Step 4: Environment Variables
Add these in Render Dashboard → Environment:
```
NODE_ENV=production
PORT=10000
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/felicity?retryWrites=true&w=majority
JWT_SECRET=your-super-secret-jwt-key-minimum-32-characters-long
JWT_EXPIRE=7d
FRONTEND_URL=https://your-project.vercel.app
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-app-password
```

**Important**: Generate a secure JWT_SECRET:
```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

#### Step 5: Deploy
- Click "Create Web Service"
- Wait for deployment (first deploy takes 2-5 minutes)
- Your API will be live at `https://your-service.onrender.com`

#### Step 6: Initialize Admin Account
After backend is deployed, run:
```bash
# Using Render Shell or local with production DB
node scripts/seedAdmin.js
```
Default credentials: `admin@felicity.com` / `admin123`

---

### 🔄 Update Frontend with Backend URL

After backend is deployed:
1. Copy your Render backend URL
2. Go to Vercel Dashboard → Settings → Environment Variables
3. Update `VITE_API_URL` to your Render URL + `/api`
   Example: `https://felicity-backend.onrender.com/api`
4. Redeploy frontend (Vercel → Deployments → Redeploy)

## 🔧 Troubleshooting

### CORS Errors
- Ensure `FRONTEND_URL` in backend matches your Vercel URL exactly
- No trailing slash in URLs

### API Connection Failed
- Verify `VITE_API_URL` in Vercel includes `/api` suffix
- Check backend is running (visit base URL)
- Check browser console for exact error

### Database Connection Failed  
- Verify MongoDB connection string is correct
- Ensure IP whitelist includes `0.0.0.0/0`
- Check database user permissions

### Build Failures
**Frontend**:
- Check Node version compatibility (v18+ recommended)
- Verify all dependencies installed
- Check for console errors in build logs

**Backend**:
- Verify `package.json` has correct start script
- Check environment variables are set
- Review Render build logs

2. Connect GitHub repository
3. Set root directory to `backend`
4. Set build command: `npm install`
5. Set start command: `npm start`
6. Add environment variables:
   - `MONGODB_URI`: Your MongoDB Atlas connection string

   - `JWT_EXPIRE`: Token expiration (default: 7d)
   - `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_USER`, `EMAIL_PASS`: Email configuration
   - `FRONTEND_URL`: Your frontend URL
   - `NODE_VERSION`: 18 (or latest LTS)
7. Deploy

1. Create new project on Railway
2. Connect GitHub repository
3. Add environment variables (same as above)
4. Railway will auto-detect Node.js and deploy

git push heroku main

heroku config:set JWT_SECRET="your_secret"

## Troubleshooting

**CORS Errors:**
- Ensure `FRONTEND_URL` in backend matches your deployed frontend URL exactly
- Check CORS configuration in `backend/server.js`

**Database Connection:**
- Verify MongoDB Atlas connection string
- Check IP whitelist (should allow 0.0.0.0/0 for cloud deployment)
- Ensure database user has read/write permissions

**Email Not Sending:**
- Verify email credentials
- For Gmail, use App Password (not regular password)
- Check email settings in backend .env

**Build Failures:**
- Check Node.js version compatibility
- Ensure all dependencies are in package.json
- Check build logs for specific errors
